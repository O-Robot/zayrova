# Zayrova Project Context

## Product summary

Zayrova is a Flutter e-commerce mobile app being built as a portfolio-level product and a reusable template that can later be resold to different brands.

The goal is not to create a small demo app. The goal is to create a polished, full-commerce mobile application that feels premium, scalable, and adaptable for fashion, lifestyle, beauty, accessories, or general retail brands.

## Current status

The existing project is an unfinished Flutter UI prototype. It already contains onboarding, authentication screens, home browsing, category, product details, wishlist, location, and partial cart screens.

The app currently needs stronger architecture, real API integration, actual state management, cleaner feature separation, and a premium design flow.

## Design sources

There are two design references inside the `docs` folder:

- `docs/old/`
  - This represents the earlier design direction.
  - It can be used for reference only.
  - Do not copy it blindly.

- `docs/new/`
  - This represents the new preferred flow and should guide the final product experience.
  - Use this as the primary reference for screen order, user journey, and app completeness.

## Product direction

Use the new design flow as the main direction.

Use the old design only when it has useful patterns that fit the final design system.

The final app should feel like a complete e-commerce template with:

- onboarding
- sign in
- sign up
- forgot password
- product discovery
- product search
- product filters
- categories
- product details
- wishlist
- cart
- checkout
- delivery address
- payment selection
- order confirmation
- order history
- order tracking
- notifications
- profile
- settings
- help and support
- legal/static pages

## Important rule

Do not downgrade the project into a simple UI clone. Zayrova must be built as a real, maintainable Flutter application with proper API calls, models, repositories, state management, and reusable components.
