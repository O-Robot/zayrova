# Feature Scope

## Target

Zayrova should become a full e-commerce Flutter application template.

It should be complete enough to impress as a portfolio project and structured well enough to be adapted or resold.

## Core features

### 1. Onboarding

Required:

- splash screen
- onboarding slides
- get started flow
- onboarding persistence

### 2. Authentication

Required:

- sign in
- sign up
- forgot password
- OTP or verification UI
- reset password
- complete profile
- session persistence
- logout

Temporary auth should use a free API where possible.

### 3. Home

Required:

- location/header area
- search entry
- promotional banner
- categories
- featured products
- popular products
- new arrivals or recommended products
- bottom navigation

### 4. Product discovery

Required:

- product listing
- category listing
- category products
- search results
- filters
- sorting
- empty state
- loading state
- error state

### 5. Product details

Required:

- image gallery
- product title
- price
- discount where available
- rating/reviews summary
- description
- variant selection where applicable
- quantity selection
- wishlist toggle
- add to cart

### 6. Wishlist

Required:

- add to wishlist
- remove from wishlist
- wishlist list
- empty wishlist state

### 7. Cart

Required:

- cart item list
- quantity increase/decrease
- remove item
- subtotal
- delivery fee placeholder
- total
- checkout button
- empty cart state

### 8. Checkout

Required:

- shipping address
- delivery method
- payment method
- order summary
- place order
- payment success screen
- payment failed screen

### 9. Orders

Required:

- order history
- order details
- order tracking
- order status timeline
- review/rating screen where supported

### 10. Notifications

Required:

- notification list
- empty notification state
- notification detail where needed

### 11. Profile and settings

Required:

- profile overview
- edit profile
- addresses
- payment methods placeholder
- security settings
- app settings
- help/support
- privacy policy
- terms and conditions

## Out of scope for first build

These can be added later:

- real payment gateway
- real push notifications
- live chat backend
- admin dashboard
- vendor management
- coupon engine
- real inventory management

## Quality requirement

Every feature should be connected to real app state. Avoid beautiful dead screens.
